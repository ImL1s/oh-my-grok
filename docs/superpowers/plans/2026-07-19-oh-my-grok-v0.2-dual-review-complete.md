### Task 4: frozen acceptance runner + prd schema (C4 + I3)

**Files:**
- Create: `omg_cli/acceptance.py`, `tests/test_acceptance.py`
- Modify: `omg_cli/modes.py`, `omg_cli/state.py`, `omg_cli/main.py` (optional `omg accept` / internal)

**Schema (prd / acceptance manifest):**
```json
{
  "version": 1,
  "goal": "...",
  "stories": [
    {"id": "s1", "title": "...", "commands": [["pytest", "tests/test_foo.py", "-q"]]}
  ],
  "global_commands": [["pytest", "tests/", "-q"]]
}
```
- Commands MUST be argv arrays (no bare shell strings by default).
- `validate_prd(data)` → raises `ValueError` on bad schema / empty stories without commands.
- `freeze_acceptance(root, run_id, prd)` → write `runs/<id>/acceptance.manifest.json` + `acceptance.sha256`.
- `run_acceptance(root, run_id)` → execute frozen commands with timeout, cwd=root, sanitized env (no OMG_ALLOW_EXTERNAL_CLI injection from model); write `acceptance.result.json` with exit codes; return ok bool.
- `set_verified` requires `acceptance.result.json` with `"passed": true` AND matching sha of manifest (not agent-forged arbitrary passed:true alone). Prefer: only `run_acceptance` may write `acceptance.result.json` with CLI stamp field `"writer": "omg-cli"`.

**Ralph loop change:**
1. After each iter (or dry_run skip exec): if prd has stories+commands, freeze if not frozen; run_acceptance; only then set_verified.
2. Without valid acceptance commands → never verified; exit non-zero for ralph when `--require-acceptance` (default True for ralph).
3. Inject fresh context pack: story id, acceptance argv list, last failure tail path.

**Tests:** forge `{passed:true}` without writer stamp → set_verified fails; CLI runner stamp → ok; bad prd schema fails.

**Commit:** `feat: frozen acceptance runner and prd schema gate`

### Task 5: ULW convergence protocol (C6)

**Files:**
- Create: `omg_cli/integrate.py`, `tests/test_integrate.py`
- Modify: `omg_cli/main.py`, `skills/omg-ultrawork/SKILL.md`, `README.md`

**Behavior:**
1. `preflight_clean_tree(root)` → `git status --porcelain` empty or fail (no auto-stash).
2. Child result envelope schema under `.omg/artifacts/ulw-results/<task_id>.json`:
   ```json
   {"task_id":"...","base_sha":"...","head_sha":"...","worktree_path":"...","changed_files":[],"status":"ok|failed","evidence":"..."}
   ```
3. `integrate_results(root, run_id)`:
   - sort by task_id
   - for each ok envelope: verify base_sha matches recorded run base; apply via `git -C worktree` merge strategy **primary: cherry-pick head_sha onto leader** or `git merge --no-ff` from worktree branch if present
   - on conflict: stop, status failed, leave worktrees
   - after all: optional run_acceptance on leader
4. CLI: `omg integrate [--run ID]` and `omg ulw` records `base_sha` at create_run.

**v0.2 minimal integrator:** if envelopes missing, dry document path; implement real merge for envelopes present with valid worktree_path + head_sha using `git cherry-pick` or merge; unit test with temp git repos.

**Commit:** `feat: ULW clean-tree preflight and result envelope integrator`
